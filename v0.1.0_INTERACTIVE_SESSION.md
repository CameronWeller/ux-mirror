#### Stop Analysis
- Click "Stop Analysis" when done

- Results are saved

### 3. CLI Commands (Alternative)

```bash

ux-tester test --after

# Analyze specific image
ux-tester analyze --image screenshot.png

# List all captures
ux-tester list

# Clean old captures (keep last 10)
ux-tester clean --keep 10
```

## Interactive Session Workflow

### Example: Analyzing Notepad

1. **Open Notepad** (or any application)

2. **Launch UX-MIRROR**
   ```bash
   python ux_mirror_launcher.py
   ```

3. **In the GUI:**
   - Select "Notepad" from the application dropdown
   - Click "Start Analysis"
   - Watch the log window for real-time feedback

4. **What Happens:**
   - Screenshot is captured
   - UI elements are detected
   - AI analyzes the screenshot
   - Results appear in the log
   - Recommendations are provided

## Tips for Best Results

### 1. Set API Key First
```powershell
$env:ANTHROPIC_API_KEY = "your-key-here"
```
This enables AI-powered analysis.

### 2. Use Clear Applications
- Applications with visible UI work best
- Avoid minimized or hidden windows
- Full-screen apps work great

### 3. Review Logs
- Check the log window for detailed feedback
- Look for specific recommendations
- Note any warnings or errors

### 4. Save Results
- Results are automatically saved
- Check `ux_captures/` folder
- JSON files contain full analysis

## Troubleshooting

### GUI Won't Start
```bash
# Check dependencies
pip install tkinter psutil

# Try direct launch
python ux_mirror_launcher.py
```

### No Applications Detected
- Make sure applications are running
- Check if applications are visible (not minimized)
- Try refreshing the application list

### API Key Issues
```bash
# Verify key is set
python tests/test_api_key_validation.py

# Set key in PowerShell
$env:ANTHROPIC_API_KEY = "your-key"
```

### Analysis Not Working
- Check API key is set
- Verify internet connection (for AI analysis)
- Check log window for error messages
